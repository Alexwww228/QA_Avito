# Тест-кейсы для API микросервиса объявлений (Avito QA Internship)

**Host:** `https://qa-internship.avito.com`  
**Версия API в коллекции:** `api/1` (основные 4 ручки задания)

## Контракт (по Postman-коллекции)

- **POST** `/api/1/item` — создать объявление  
  - request body: `sellerID` (int), `name` (string), `price` (int), `statistics` (object)
  - response body: объект объявления с `id` (string UUID), `sellerId`, `name`, `price`, `statistics`, `createdAt`
- **GET** `/api/1/item/{id}` — получить объявление по идентификатору  
  - response body: **массив** объявлений (обычно 1 элемент)
- **GET** `/api/1/{sellerID}/item` — получить все объявления продавца  
  - response body: **массив** объявлений
- **GET** `/api/1/statistic/{id}` — получить статистику по объявлению  
  - response body: **массив** объектов статистики (обычно 1 элемент)

## Техники тест-дизайна использованные в тест-кейсах

1. **Анализ граничных значений (Boundary Value Analysis)**
   - Минимальные и максимальные значения для price, sellerId
   - Граничные значения для ID (0, -1, максимальное значение)

2. **Классы эквивалентности (Equivalence Partitioning)**
   - Валидные и невалидные типы данных
   - Существующие и несуществующие ID

3. **Тестирование состояний (State Transition Testing)**
   - Жизненный цикл объявления (создание → получение → статистика)

4. **Pairwise Testing**
   - Комбинации различных полей объявления

5. **Негативное тестирование**
   - Отсутствующие обязательные поля
   - Невалидные типы данных
   - Некорректный JSON

6. **Тестирование идемпотентности**
   - Повторные запросы создания

7. **Исследовательское тестирование**
   - Unicode символы
   - Спецсимволы
   - Очень длинные строки
